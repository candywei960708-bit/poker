# 系統功能說明

## 一、系統目的

本系統以 C++ 物件導向程式設計實作「德州撲克（Texas Hold’em）」的簡化版本，透過類別分工與模組化設計，模擬撲克牌遊戲流程，並達到以下目標：

熟悉物件導向設計（OOP）,
實作封裝（Encapsulation）,
落實權責分離（Single Responsibility Principle）,
建立可擴充的遊戲架構。

## 二、系統架構

### 1️. Card（撲克牌類別）

功能：

表示單一張撲克牌
儲存點數與花色

主要功能：

建立牌物件（例如：A♠）
提供點數與花色查詢
將牌轉換為可讀字串

### 2️. Deck（牌堆類別）

功能：

管理整副 52 張撲克牌

主要功能：

初始化完整牌組
洗牌（Shuffle）
發牌（Deal）
每次發牌會從牌堆中移除該張牌

### 3️. Player（玩家類別）

功能：

管理玩家手牌

主要功能：

接收發牌
儲存手牌
顯示手牌內容

### 4️. HandEvaluator（牌型判斷類別）

功能：

判斷玩家手牌強度

目前版本（簡化）：

只比較最大點數

可擴充：

一對（Pair）
兩對（Two Pair）
同花（Flush）
順子（Straight）
葫蘆（Full House）
同花順（Straight Flush）

### 5️. Game（遊戲控制類別）

功能：

控制整個遊戲流程

主要流程：

初始化牌堆
洗牌
發牌給玩家
顯示手牌
計算分數
判斷勝負
此類別是整個系統的核心控制器

## 6️. main（主程式）

功能：

程式進入點
建立 Game 物件並啟動遊戲

## 三、系統流程

開始程式
   ↓
建立 Game 物件
   ↓
初始化 Deck（52張牌）
   ↓
洗牌
   ↓
發牌給玩家（每人2張）
   ↓
顯示手牌
   ↓
計算牌力
   ↓
比較勝負
   ↓
輸出結果
   ↓
結束程式
